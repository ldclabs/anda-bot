# Terminal

Après avoir exécuté `anda`, l'interface utilisateur du terminal se connecte au démon local. Lorsqu'une conversation se termine avec succès, elle est soumise au Brain en arrière-plan pour la formation de la mémoire à long terme.

## Contrôles de saisie

| Touche | Action |
| --- | --- |
| Entrée | Envoyer le message actuel |
| Maj+Entrée | Insérer un saut de ligne |
| Ctrl+U | Vider la zone de saisie |
| Ctrl+A / Ctrl+E | Aller au début ou à la fin de la saisie |
| Échap | Afficher le statut |
| Ctrl+C | Quitter |

Après avoir modifié les fournisseurs de modèles dans `~/.anda/config.yaml`, actualisez les modèles depuis le lanceur ou le panneau latéral du navigateur, ou exécutez :

```bash
anda models reload
```

Après avoir modifié des paramètres du démon nécessitant un redémarrage, tapez :

```text
/reload
```

## Commandes Slash

| Commande | Quand l'utiliser |
| --- | --- |
| `/goal message` | Place un objectif dans la boucle de tâche à long terme jusqu'à ce que le vérificateur décide qu'il est terminé ou nécessite un travail supplémentaire. |
| `/loop message` | Exécute un prompt ou une commande slash de manière répétée. Décrivez un intervalle en langage naturel si vous le souhaitez, ou omettez-le pour laisser le modèle définir son propre rythme. |
| `/side message` ou `/btw message` | Démarre une tâche secondaire pour de la recherche, de l'inspection ou du travail qui ne doit pas interrompre la conversation principale. |
| `/skill nom-skill message` | Demande à une compétence (Skill) de runtime spécifique de gérer la requête. |
| `/steer message` | Interrompt le prochain appel d'outil et oriente la réponse actuelle avec de nouvelles instructions. |
| `/stop message` | Arrête la tâche actuelle et laisse la conversation inactive ; le message peut inclure une raison. |
| `/cancel message` | Annule la session de conversation active ; le message peut inclure une raison. |

Exemples :

```text
/goal Terminer cette préparation de version : lancer les tests, vérifier le journal des modifications, valider les docs du script d'install et résumer les risques.
/loop toutes les 5 minutes /side Vérifier la file d'attente de déploiement et résumer les bloqueurs.
/side Vérifier si cette mise à niveau de dépendance présente un risque de compatibilité.
/skill git-assistant Rédiger un message de commit à partir du diff actuel.
/steer Arrêter la refactorisation et corriger uniquement l'erreur de build d'abord.
```

## Commandes CLI courantes

| Commande | Objectif |
| --- | --- |
| `anda` | Ouvrir l'interface utilisateur interactive du terminal. |
| `anda update` | Mettre à jour une version installée par le script d'installation. |
| `anda status` | Indiquer si le démon d'arrière-plan est en cours d'exécution. |
| `anda start` | Démarrer le démon d'arrière-plan. |
| `anda stop` | Arrêter le démon d'arrière-plan. |
| `anda restart` | Redémarrer le démon d'arrière-plan. |
| `anda models reload` | Recharger les fournisseurs de modèles depuis `config.yaml` sans redémarrer le démon. |
| `anda autostart status` | Indiquer si le démarrage automatique à la connexion est enregistré. |
| `anda agent run --prompt "..."` | Envoyer une requête et attendre le résultat complet sans ouvrir l'interface utilisateur. |
| `anda voice --record-secs 8` | Démarrer la saisie vocale ; requiert la configuration de la transcription. |

La lecture vocale requiert également `tts.enabled: true`. Si vous souhaitez uniquement une entrée vocale et une sortie textuelle, ajoutez `--no-playback`.
