# Extension de navigateur

L'extension de navigateur Anda Bot ouvre Anda dans le panneau latéral natif des navigateurs basés sur Chromium — Google Chrome et Microsoft Edge. Elle sert de pont vers le démon local Anda qui s'exécute sur votre propre ordinateur, afin que le contexte du navigateur, les outils de terminal, les fichiers de projet, les compétences (Skills), les sous-agents, les canaux et la mémoire du Brain restent regroupés dans un seul runtime.

## Prérequis

- Google Chrome ou Microsoft Edge 116 ou supérieur (ou tout autre navigateur basé sur Chromium).
- Anda installé et configuré avec au moins une clé API de fournisseur de modèle.
- Le démon local Anda en cours d'exécution sur l'ordinateur que vous contrôlez.

Installez l'extension à partir de la boutique de modules complémentaires de votre navigateur :

- [Anda Bot sur le Chrome Web Store](https://chromewebstore.google.com/detail/anda-bot/injpfajmddchcphfkdkiflfddmajglfd) — Google Chrome et autres navigateurs Chromium.
- [Anda Bot sur Microsoft Edge Add-ons](https://microsoftedge.microsoft.com/addons/detail/anda-bot/hljillhnmfbobihkehdlpmhbmdgophah) — Microsoft Edge.

La même extension et les mêmes étapes de configuration s'appliquent aux deux navigateurs.

## Connecter le panneau latéral

Démarrez le démon local :

```bash
anda start
```

Générez un jeton de navigateur :

```bash
anda browser token --days 365
```

Copiez l'URL de la passerelle (Gateway URL) et le Bearer token affichés. Ouvrez le panneau latéral Anda Bot dans Chrome ou Edge, accédez aux paramètres, collez les deux valeurs et enregistrez. Une fois la connexion établie, vous pouvez discuter avec le même agent local depuis n'importe quelle page Web.

## Travailler depuis une page

Lorsque vous envoyez un message depuis le panneau latéral, Anda reçoit le titre et l'URL de l'onglet actif comme contexte de requête. Si vous demandez un travail sur le navigateur, l'agent peut utiliser des actions de navigation pour inspecter les pages et manipuler les onglets.

Exemples d'utilisation :

- Résumer ou comparer la page que vous lisez.
- Collecter des informations sur plusieurs onglets tout en conservant les URL sources associées.
- Remplir des formulaires, cliquer sur des boutons, faire défiler, prendre des captures d'écran ou exporter une page au format PDF lorsque la tâche le requiert.
- Préparer des commandes de shell ou inspecter des fichiers de projet locaux tout en conservant le contexte de la page de navigation.
- Ajouter des messages importants de l'assistant aux favoris, les organiser dans des dossiers et revenir à la conversation d'origine plus tard.

## Fonctionnalités du navigateur

L'extension expose des outils de navigation au démon local. Les actions courantes incluent :

- Lister, changer, ouvrir, fermer, recharger et naviguer dans les onglets.
- Lire les sélections de texte, extraire le texte de la page, capturer des captures d'écran, inspecter les données d'accessibilité et imprimer les pages en PDF.
- Cliquer, saisir du texte, appuyer sur des touches, faire défiler, sélectionner dans des listes déroulantes, téléverser des fichiers et gérer les boîtes de dialogue.
- Télécharger des fichiers et inspecter les téléchargements initiés par le navigateur.

Les actions de page et de saisie ciblent l'onglet actif. Si la tâche nécessite un autre onglet, demandez d'abord à Anda de changer d'onglet.

Pour ouvrir des fichiers ou des dossiers locaux à l'aide de `open_file`, cochez la case `Autoriser l'accès aux URL de fichier` dans la page des détails de l'extension Anda Bot du navigateur (dans Edge, ouvrez `edge://extensions` → Anda Bot → Détails).

## Données et runtime locaux

L'extension stocke l'URL de la passerelle, le Bearer token, l'ID de session de navigation et les paramètres dans le stockage local du navigateur. Elle transmet les prompts, les pièces jointes sélectionnées, les métadonnées de session de navigation et les informations d'onglet pertinentes au démon Anda auquel vous vous connectez.

Le démon et les fournisseurs de modèles configurés dans votre installation Anda peuvent traiter le prompt et les données de tâche. N'utilisez l'extension qu'avec un démon et une configuration de modèle de confiance, en particulier sur les pages sensibles.

## Canaux distants

Le panneau latéral du navigateur n'est qu'une interface parmi d'autres. Si vous configurez des canaux WeChat, Telegram, Discord ou Lark / Feishu, les messages de ces partenaires peuvent atteindre le même démon local pendant qu'il continue de s'exécuter sur votre ordinateur.

Reportez-vous à la section [Canaux et voix](../runtime/channels) pour la configuration des canaux.

## Dépannage

- Si le panneau latéral ne parvient pas à se connecter, confirmez que `anda start` est toujours actif et que la Gateway URL et le Bearer token ont été saisis exactement.
- Si les actions de navigation échouent sur des pages système protégées telles que les URL `chrome://` ou `edge://`, ou sur la boutique de modules du navigateur, ouvrez une page Web classique. Les navigateurs basés sur Chromium bloquent les scripts d'extension sur les pages protégées.
- Si une erreur de modèle survient, vérifiez la clé de votre fournisseur de modèle dans `~/.anda/config.yaml` ou la variable d'environnement utilisée pour démarrer le démon.
