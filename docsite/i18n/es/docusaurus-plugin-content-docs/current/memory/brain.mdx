# Cerebro de memoria Brain

Anda Brain es el sistema de memoria a largo plazo que impulsa a Anda Bot. No es una base de datos vectorial simple ni un archivo Markdown en constante crecimiento; es un cerebro de memoria diseñado específicamente para agentes de IA.

El Brain convierte las conversaciones, los archivos, los resultados de las herramientas y el contexto de los canales en un Cognitive Nexus (nexo cognitivo) en constante crecimiento: personas, proyectos, preferencias, eventos, decisiones, líneas de tiempo y las relaciones entre ellos.

## Bucle de tres fases

### Formación (Formation)

La Formación codifica conversaciones útiles y resultados de trabajo en memorias estructuradas. No solo se pregunta si apareció una frase en el chat; pregunta quién la dijo, a qué proyecto pertenece, qué preferencia revela, qué decisión se tomó y cuándo sucedió.

### Recuerdo (Recall)

El Recuerdo permite consultar el grafo de memoria utilizando lenguaje natural. Un buen recuerdo proporciona el contexto necesario para la tarea actual, en lugar de limitarse a devolver fragmentos de texto similares o cercanos.

### Mantenimiento (Maintenance)

El Mantenimiento consolida los fragmentos de información, fusiona los datos duplicados, reduce la influencia del conocimiento obsoleto y preserva las líneas de tiempo cuando cambian los hechos.

## Cómo enseñar a Anda

No es necesario mantener manualmente archivos de memoria. Un mejor enfoque consiste en indicarle claramente a Anda las preferencias, los hechos estables y los cambios que ocurran.

```text
Recuerda: me gustan las notas de lanzamiento cortas, pero conserva una sección de riesgos.
El proveedor de modelos predeterminado para este espacio de trabajo cambió de A a B.
Cuando mencionemos a Alice, nos referimos a la diseñadora del equipo móvil.
¿Recuerdas los antecedentes del proyecto de migración de pagos?
```

## Qué hace que una memoria sea buena

- Contexto: el proyecto, la persona, el archivo, el canal o el rango de tiempo deben estar claros.
- Evolución: cuando cambie una preferencia, indique tanto la versión antigua como la nueva.
- Límites: la información temporal se puede marcar como válida únicamente para la tarea actual.
- Corrección: si Anda recuerda algo incorrectamente, indíquele directamente la versión correcta.

## Datos locales y proveedores de modelos

Los datos del grafo de memoria se almacenan localmente en `~/.anda/db` de forma predeterminada. Los proveedores de modelos pueden seguir participando en la formación de la memoria, el soporte de recuerdo y la síntesis, así que elija los proveedores y puntos de conexión (endpoints) que mejor se adapten a sus necesidades de privacidad.

Para priorizar un proveedor para tareas relacionadas con la memoria, agregue la etiqueta `brain`:

```yaml
model:
  providers:
    - family: anthropic
      model: "deepseek-v4-pro"
      api_base: "https://api.deepseek.com/anthropic"
      api_key: "TU_CLAVE_API"
      labels: ["primary", "brain"]
```
