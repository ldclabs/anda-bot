# Trabajo de largo alcance y subagentes

Anda Bot está diseñado para objetivos complejos que necesitan continuidad. Conserva el objetivo, revisa progreso, compacta contexto y continúa cuando hace falta.

## Usar /goal

```text
/goal Termina este release: tests, changelog, docs de instalación y resumen de riesgos.
```

`/goal` abre un ciclo de largo alcance. Cuando el agente queda libre, el verificador decide si el resultado está completo o necesita continuar.

## Usar /loop

`/loop` ejecuta repetidamente un prompt o comando slash. Describe el intervalo en la solicitud con el lenguaje que te resulte natural, como `cada 5 minutos`, `5m` o unidades localizadas. El modelo interpreta el intervalo y puede crear un cron job recurrente cuando haga falta:

```text
/loop cada 5 minutos /side Revisa si la cola de despliegue está bloqueada y resume el resultado.
```

Si no describes un intervalo, Anda deja que el modelo marque el ritmo y continúe cuando haga falta:

```text
/loop Sigue revisando la migración hasta probar cada paso de validación.
```

## Usar /side

`/side` o `/btw` crea una tarea lateral para investigar, revisar o recoger contexto sin interrumpir la conversación principal.

```text
/side Comprueba si esta dependencia afecta a macOS arm64.
```

## Usar /skill

```text
/skill git-assistant Redacta un Conventional Commit desde el diff actual.
```

## Qué hacen los subagentes

Pueden investigar, implementar, revisar o supervisar. La sesión principal mantiene el objetivo y el juicio final; los subagentes hacen trabajo focalizado.

## Herramientas y workspace local

Anda incluye memoria, notas, shell, archivos, tareas, skills, cron y subagentes. Las herramientas de archivo y shell usan `~/.anda/workspace` por defecto.

Di claramente el objetivo, los criterios de aceptación y los límites.

```text
/goal Arregla el build de docs. No edites Rust. Termina cuando pnpm --dir docsite build pase.
```
