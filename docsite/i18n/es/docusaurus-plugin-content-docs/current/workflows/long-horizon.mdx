# Trabajo de largo alcance y Subagentes

Anda Bot está diseñado para gestionar objetivos complejos que requieren continuidad, no solo para dar una única respuesta. El trabajo de largo alcance conserva el objetivo original, verifica el progreso realizado, compacta el contexto cuando es necesario y continúa trabajando en la tarea según se requiera.

## Usar /goal

El comando `/goal` marca una solicitud como un objetivo de largo alcance:

```text
/goal Finalizar esta preparación de lanzamiento: ejecutar pruebas, revisar el registro de cambios, verificar la documentación del script de instalación y resumir los riesgos.
```

Cuando el agente principal se encuentra inactivo, el verificador de objetivos evalúa si el resultado actual está completo. Si se requiere más trabajo, genera una tarea de seguimiento para el agente principal. Si el contexto de la conversación se vuelve demasiado extenso, Anda compacta el estado actual y abre una conversación vinculada para continuar con la misma sesión sin perder el hilo.

## Usar /loop

El comando `/loop` ejecuta un prompt o comando de barra diagonal de forma repetida. Puede describir el intervalo en la solicitud utilizando cualquier formato natural, como `every 5 minutes` (cada 5 minutos), `5m` o unidades de tiempo localizadas. El modelo interpreta el intervalo y puede crear una tarea programada cron del agente en segundo plano cuando sea necesario:

```text
/loop cada 5 minutos /side Verificar si la cola de despliegue está bloqueada y resumir el resultado.
```

Si no especifica un intervalo, Anda permite que el modelo determine su propio ritmo en el bucle y continúe cuando sea necesario realizar más trabajo:

```text
/loop Continuar verificando la migración hasta que se demuestre que cada paso de validación se ha completado con éxito.
```

## Usar /side

Los comandos `/side` o `/btw` colocan una solicitud en una tarea secundaria independiente. Resulta muy útil para:

- Investigar un riesgo específico.
- Solicitar que otra perspectiva o rol revise la conclusión actual.
- Recopilar información de contexto sin interrumpir la conversación principal.

```text
/side Comprobar si esta actualización de dependencias afecta a los usuarios de macOS arm64.
```

## Usar /skill

Las habilidades (Skills) de runtime personales residen en `~/.anda/skills`, mientras que las habilidades integradas en las versiones oficiales se cargan desde `~/.anda/bundled-skills`. Cuando desee que un flujo de trabajo específico funcione con una solicitud, use:

```text
/skill git-assistant Redactar un mensaje de commit de tipo Conventional Commit a partir del diff actual.
```

## Qué hacen los subagentes

Los subagentes pueden asumir roles especializados como investigación, implementación, revisión y supervisión. La sesión principal conserva el objetivo general y el juicio final, mientras que los subagentes se encargan de llevar a cabo el trabajo enfocado.

Esta estructura es ideal para:

- Trabajos de investigación que requieran la lectura de múltiples archivos o referencias de documentación.
- Tareas de código donde la implementación y la revisión deban estar claramente separadas.
- Objetivos que requieran verificaciones repetidas de si el trabajo se ha completado realmente.
- Trabajos que requieran el uso de herramientas externas sin perder el hilo de la conversación principal.

## Herramientas y espacio de trabajo local

Las herramientas básicas de Anda include memoria, notas, comandos de shell, lectura/escritura de archivos, lista de tareas (todo), habilidades, tareas programadas (cron) y subagentes. Las herramientas de archivos y shell operan de forma predeterminada en `~/.anda/workspace`.

Cuando una tarea involucre al repositorio actual o a herramientas externas, indique claramente el objetivo, los criterios de aceptación y los límites de la tarea.

```text
/goal Corregir la compilación de la documentación en el repositorio actual. No edite código Rust. Completado significa que pnpm --dir docsite build pase con éxito.
```

## Cuándo no usar /goal o /loop

Las preguntas y respuestas simples, las explicaciones puntuales y las búsquedas rápidas de comandos no requieren el uso de `/goal` o `/loop`. Para estos casos, envíe un mensaje normal. Reserve `/goal` para trabajos que requieran auditoría de finalización y `/loop` para seguimientos recurrentes o tareas autodirigidas.
