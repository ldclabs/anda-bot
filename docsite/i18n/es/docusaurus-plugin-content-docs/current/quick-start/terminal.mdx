# Terminal

Al ejecutar `anda`, la interfaz de usuario de la terminal se conecta al daemon local. Cuando una conversación se completa con éxito, se envía a Brain en segundo plano para la formación de memoria a largo plazo.

## Controles de entrada

| Tecla | Acción |
| --- | --- |
| Enter | Enviar el mensaje actual |
| Shift+Enter | Insertar un salto de línea |
| Ctrl+U | Limpiar el cuadro de entrada |
| Ctrl+A / Ctrl+E | Moverse al inicio o al final del texto de entrada |
| Esc | Ver estado |
| Ctrl+C | Salir |

Después de editar los proveedores de modelos en `~/.anda/config.yaml`, actualice los modelos desde el lanzador o el panel lateral del navegador, o ejecute:

```bash
anda models reload
```

Si ha modificado parámetros del daemon que requieren un reinicio, escriba:

```text
/reload
```

## Comandos de barra diagonal (Slash Commands)

| Comando | Cuándo usarlo |
| --- | --- |
| `/goal mensaje` | Coloca un objetivo en el bucle de largo alcance hasta que el verificador determine que está completo o requiere más trabajo. |
| `/loop mensaje` | Ejecuta un prompt o un comando de barra diagonal de forma repetida. Puede describir un intervalo en lenguaje natural (ej. `every 5 minutes`) o dejarlo vacío para que el modelo marque su propio ritmo. |
| `/side mensaje` o `/btw mensaje` | Inicia una tarea secundaria para investigación, inspección o trabajo que no debe interrumpir la conversación principal. |
| `/skill nombre-habilidad mensaje` | Solicita a una habilidad (Skill) de runtime específica que gestione la solicitud. |
| `/steer mensaje` | Interrumpe la próxima llamada a herramienta y dirige la respuesta actual con nuevas instrucciones. |
| `/stop mensaje` | Detiene la tarea actual y deja la conversación inactiva; el mensaje puede incluir un motivo. |
| `/cancel mensaje` | Cancela la sesión de conversación activa; el mensaje puede incluir un motivo. |

Ejemplos:

```text
/goal Finalizar esta lista de preparación de lanzamiento hasta que se confirmen las pruebas y el registro de cambios.
/loop cada 5 minutos /side Verificar la cola de despliegue y resumir los bloqueos.
/side Comprobar si esta actualización de dependencia tiene riesgos de compatibilidad.
/skill git-assistant Redactar un mensaje de commit a partir del diff actual.
/steer Detener la refactorización y solucionar primero únicamente el error de compilación.
```

## Comandos comunes de la CLI

| Comando | Propósito |
| --- | --- |
| `anda` | Abre la interfaz de usuario de la terminal interactiva. |
| `anda update` | Actualiza una versión instalada mediante el script de instalación. |
| `anda status` | Muestra si el daemon en segundo plano se está ejecutando. |
| `anda start` | Inicia el daemon en segundo plano. |
| `anda stop` | Detiene el daemon en segundo plano. |
| `anda restart` | Reinicia el daemon en segundo plano. |
| `anda models reload` | Recarga los proveedores de modelos desde `config.yaml` sin reiniciar el daemon. |
| `anda autostart status` | Muestra si el inicio automático al iniciar sesión está registrado. |
| `anda agent run --prompt "..."` | Envía una solicitud y espera el resultado completo sin abrir la interfaz de usuario. |
| `anda voice --record-secs 8` | Inicia la entrada de voz; requiere la configuración de transcripción. |

La reproducción de voz también requiere que `tts.enabled: true` esté configurado. Si solo desea entrada por voz y salida de texto, añada `--no-playback`.
