# Extensión del navegador

La extensión del navegador Anda Bot abre Anda en el panel lateral nativo de los navegadores basados en Chromium: Google Chrome y Microsoft Edge. Sirve como puente hacia el daemon local de Anda que se ejecuta en su propia computadora, de modo que el contexto del navegador, las herramientas de terminal, los archivos del proyecto, las habilidades (Skills), los subagentes, los canales y la memoria del Brain permanezcan integrados en un único runtime.

## Requisitos

- Google Chrome o Microsoft Edge 116 o posterior (o cualquier navegador basado en Chromium).
- Anda instalado y configurado con al menos una clave API de proveedor de modelos.
- El daemon local de Anda ejecutándose en la computadora que usted controla.

Instale la extensión desde la tienda de complementos de su navegador:

- [Anda Bot en Chrome Web Store](https://chromewebstore.google.com/detail/anda-bot/injpfajmddchcphfkdkiflfddmajglfd) (para Google Chrome y otros navegadores Chromium).
- Microsoft Edge: abra **Complementos de Edge** (`edge://extensions` → "Obtener extensiones de Microsoft Edge Add-ons") y busque **Anda Bot**.

La misma extensión y los mismos pasos de configuración funcionan en ambos navegadores.

## Conectar el panel lateral

Inicie el daemon local:

```bash
anda start
```

Genere un token de navegador:

```bash
anda browser token --days 365
```

Copie la Gateway URL and el Bearer token impresos. Abra el panel lateral de Anda Bot en Chrome o Edge, acceda a la configuración, pegue ambos valores y guarde los cambios. Una vez que la conexión se realice correctamente, podrá chatear con el mismo agente local desde cualquier página web.

## Trabajar desde una página

Al enviar un mensaje desde el panel lateral, Anda recibe el título y la URL de la pestaña actual como contexto de la solicitud. Si solicita realizar un trabajo en el navegador, el agente puede utilizar acciones del navegador para inspeccionar páginas y operar pestañas.

Ejemplos útiles:

- Resumir o comparar la página que está leyendo actualmente.
- Recopilar información a través de varias pestañas y mantener adjuntas las URL de origen.
- Completar formularios, hacer clic en botones, desplazarse, tomar capturas de pantalla o exportar páginas a PDF cuando la tarea lo requiera.
- Preparar comandos de shell o inspeccionar archivos de proyectos locales mientras mantiene el contexto de la página del navegador.
- Guardar mensajes importantes del asistente en marcadores, organizarlos en carpetas y volver a la conversación de origen más tarde.

## Capacidades del navegador

La extensión expone herramientas del navegador al daemon local. Las acciones comunes incluyen:

- Listar, cambiar, abrir, cerrar, recargar y navegar entre pestañas.
- Leer selecciones de texto, extraer el texto de la página, capturar capturas de pantalla, inspeccionar datos de accesibilidad y exportar páginas a PDF.
- Hacer clic, escribir, pulsar teclas, desplazarse, seleccionar opciones de menús desplegables, subir archivos y gestionar cuadros de diálogo.
- Descargar archivos e inspeccionar descargas iniciadas por el navegador.

Las acciones de entrada y de página se dirigen a la pestaña activa. Si la tarea requiere otra pestaña, pídale a Anda que cambie de pestaña primero.

Para permitir abrir archivos o carpetas locales con `open_file`, active la opción `Permitir acceso a las URL de archivo` en la página de detalles de la extensión Anda Bot en su navegador (en Edge, abra `edge://extensions` → Anda Bot → Detalles).

## Runtime local y datos

La extensión almacena la Gateway URL, el Bearer token, el ID de sesión del navegador y los ajustes en el almacenamiento local del navegador. Envía los prompts, los archivos adjuntos seleccionados, los metadatos de sesión y la información relevante de las pestañas al daemon de Anda al que se conecta.

El daemon y los proveedores de modelos configurados en su instalación de Anda procesarán estos datos de prompts y tareas. Utilice la extensión únicamente con un daemon y una configuración de modelos en los que confíe, especialmente cuando visite páginas con información sensible.

## Canales remotos

El panel lateral del navegador es solo una de las interfaces disponibles. Si configura canales como WeChat, Telegram, Discord o Lark / Feishu, los mensajes de esas plataformas podrán llegar al mismo daemon local mientras se mantiene en ejecución en su computadora.

Consulte [Canales y voz](../runtime/channels) para conocer la configuración de los canales.

## Resolución de problemas

- Si el panel lateral no puede conectarse, confirme que `anda start` se está ejecutando y que la Gateway URL y el Bearer token se han pegado de manera exacta.
- Si las acciones del navegador fallan en páginas protegidas como las URL `chrome://` o `edge://`, o en la tienda de complementos del navegador, abra una página web normal. Los navegadores Chromium bloquean las secuencias de comandos (scripts) de las extensiones en las páginas del sistema protegidas.
- Si aparece un error del modelo, verifique la clave del proveedor de modelos en `~/.anda/config.yaml` o la variable de entorno utilizada para iniciar el daemon.
