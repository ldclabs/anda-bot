# Документация Anda Bot

Anda Bot — это локальный AI-помощник с приоритетом памяти. Десктопный лаунчер и локальная фоновая служба поддерживают работу Brain на вашем компьютере, в то время как боковая панель браузера, терминал, файлы, навыки (Skills), задачи по расписанию (cron), субагенты, каналы коммуникации и голосовые функции могут наполнять одну и ту же долгосрочную графовую память.

Эта документация предназначена для тех, кто хочет полностью владеть слоем персонального ассистента, а не арендовать память у одного аккаунта модели. Начните с установки приложения, затем подключите браузер, настройте провайдеров моделей и узнайте, как Brain переносит контекст между сессиями и моделями.

## Начните отсюда

- [Установить Anda](./quick-start/install): выберите десктопный лаунчер, Homebrew, скрипт установки, Windows PowerShell или запуск из исходного кода.
- [Связать с браузером](./quick-start/browser-extension): подключите боковую панель к локальному демону с помощью Gateway URL и Bearer-токена.
- [Использовать терминал](./quick-start/terminal): изучите отправку сообщений, многострочный ввод, `/goal`, `/loop`, `/side`, `/skill`, `/steer` и базовые команды.
- [Настроить среду выполнения](./runtime/configuration): подготовка `~/.anda/config.yaml`, провайдеров моделей, API-ключей, прокси и локальных директорий.
- [Понять память Brain](./memory/brain): изучите цикл Формирования (Formation), Вызова (Recall) и Обслуживания (Maintenance).
- [Запуск долгосрочных задач](./workflows/long-horizon): делегируйте сложные цели специализированным ролям с сохранением главной ветки разговора и финальной проверкой.
- [Подключить каналы и голос](./runtime/channels): добавьте Anda в Telegram, WeChat, Discord, Lark/Feishu и настройте голосовой ввод/вывод.

## Ментальная модель

Anda Bot — это не просто еще одно окно чата. Ключевое отличие заключается в непрерывности.

- Anda работает локально, сохраняя состояние по умолчанию в папке `~/.anda`.
- Brain превращает полезный контекст в графовую память, связывающую проекты, предпочтения, отношения, решения и временные шкалы.
- Модели и провайдеры легко заменяемы; память остается локальной и доступной для проверки.
- Боковая панель браузера, терминал, файлы, навыки, задачи cron, субагенты, каналы и голос подключаются к единой локальной среде.
- `/goal` позволяет продвигать выполнение долгосрочных целей, а `/loop` запускает повторяющиеся задачи или задачи с собственным ритмом модели.

## Кратчайший путь

```bash
brew install ldclabs/tap/anda
DEEPSEEK_API_KEY=ВАШ_API_KEY anda
```

Если вы работаете не на macOS или предпочитаете использовать лаунчер или скрипт установки, перейдите к разделу [Установить Anda](./quick-start/install).

:::tip
При первом запуске Anda автоматически создает файл `~/.anda/config.yaml`. Если в интерфейсе терминала сообщается о недостающих полях моделей, отредактируйте этот файл, сохраните его, а затем обновите модели из лаунчера, боковой панели браузера или выполнив команду `anda models reload`.
:::
