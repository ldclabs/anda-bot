# Документация Anda Bot

> Born of panda. Awakened as Anda.

Я Anda Bot, open-source Rust-агент, который работает в локальном терминале. Я запоминаю полезную работу, использую инструменты на вашей машине, координирую сабагентов и продолжаю длинные цели через границы контекста.

Документация покрывает установку, настройку моделей, команды терминала, память Hippocampus, каналы и длинные workflow.

## С чего начать

- [Установить Anda](./quick-start/install): Homebrew, скрипт, PowerShell или исходный код.
- [Терминал](./quick-start/terminal): многострочный ввод, `/goal`, `/side`, `/skill`, `/steer` и команды CLI.
- [Настроить runtime](./runtime/configuration): `~/.anda/config.yaml`, провайдеры, API-ключи, proxy и локальные директории.
- [Понять память](./memory/hippocampus): Formation, Recall и Maintenance.
- [Длинные задачи](./workflows/long-horizon): делегирование сложных целей без потери главного потока.
- [Каналы и голос](./runtime/channels): IRC, Telegram, WeChat, Discord, Lark/Feishu и голос.

## Ментальная модель

Anda не просто еще одно окно чата. Главное отличие — непрерывность.

- Он работает локально и хранит состояние в `~/.anda`.
- Может использовать shell, файлы, заметки, задачи, skills, cron и внешние инструменты.
- Anda Hippocampus превращает полезный контекст в графовую память.
- `/goal` запускает цикл, который проверяет прогресс и продолжает при необходимости.
- Терминал, каналы и голос могут делить одну линию памяти.

## Самый короткий путь

```bash
brew install ldclabs/tap/anda
DEEPSEEK_API_KEY=YOUR_API_KEY anda
```

Если вы не на macOS или предпочитаете скрипт, перейдите к [установке Anda](./quick-start/install).

:::tip
При первом запуске Anda создает `~/.anda/config.yaml`. Если UI сообщает о пропущенных полях модели, отредактируйте файл и нажмите Enter для перезагрузки.
:::
