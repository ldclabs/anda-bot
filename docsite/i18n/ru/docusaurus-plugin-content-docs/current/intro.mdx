# Документация Anda Bot

Anda Bot - локальный AI-помощник с memory-first подходом. Desktop launcher и local daemon держат Brain на вашей машине, а browser side panel, terminal, files, skills, cron, subagents, channels и voice могут питать одну graph memory.

Эта документация для тех, кто хочет владеть assistant layer, а не арендовать память у одного model account. Начните с app install, подключите браузер, настройте providers и изучите, как Brain сохраняет контекст portable между sessions и models.

- [Подключить браузер](./quick-start/browser-extension): соедините side panel с local daemon через Gateway URL и Bearer token.

- [Настроить runtime](./runtime/configuration): `~/.anda/config.yaml`, провайдеры, API-ключи, proxy и локальные директории.
- [Понять память](./memory/brain): Formation, Recall и Maintenance.
- [Длинные задачи](./workflows/long-horizon): делегирование сложных целей без потери главного потока.
- [Каналы и голос](./runtime/channels): Telegram, WeChat, Discord, Lark/Feishu и голос.

## Ментальная модель

Anda не просто еще одно окно чата. Главное отличие — непрерывность.

- Он работает локально и хранит состояние в `~/.anda`.
- Brain превращает полезный контекст в graph memory для projects, preferences, relationships, decisions и timelines.
- Models и providers можно менять; memory остается local и inspectable.
- Browser, terminal, files, skills, cron, subagents, channels и voice подключаются к одному runtime.
- `/goal` поддерживает long-horizon work, сохраняя main thread и final verification.

## Самый короткий путь

```bash
brew install ldclabs/tap/anda
DEEPSEEK_API_KEY=YOUR_API_KEY anda
```

Если вы не на macOS или предпочитаете launcher или script, перейдите к [установке Anda](./quick-start/install).

:::tip
При первом запуске Anda создает `~/.anda/config.yaml`. Если UI сообщает о пропущенных полях модели, отредактируйте файл и нажмите Enter для перезагрузки.
:::
