# Терминал

После запуска программы `anda` текстовый интерфейс терминала подключается к локальному демону. После успешного завершения сессии диалога вся история разговора передается в фоновом режиме в Brain для формирования долгосрочной памяти.

## Элементы управления вводом

| Клавиша | Действие |
| --- | --- |
| Enter | Отправить текущее сообщение |
| Shift+Enter | Вставить перенос строки |
| Ctrl+U | Очистить поле ввода |
| Ctrl+A / Ctrl+E | Перейти в начало или в конец текста ввода |
| Esc | Показать статус работы |
| Ctrl+C | Выход из программы |

После внесения изменений в настройки провайдеров моделей в файле `~/.anda/config.yaml` обновите конфигурацию из лаунчера или боковой панели браузера, либо выполните команду:

```bash
anda models reload
```

Если вы изменили системные настройки службы, требующие перезапуска демона, введите в поле чата:

```text
/reload
```

## Команды со слэшем (Slash Commands)

| Команда | Когда использовать |
| --- | --- |
| `/goal сообщение` | Переводит задачу в режим циклического контроля долгосрочной цели до тех пор, пока система проверки не подтвердит ее выполнение. |
| `/loop сообщение` | Запускает периодическое циклическое выполнение промпта или команды. Можно указать интервал на естественном языке (например, `every 5 minutes`) или опустить его для автовыбора темпа моделью. |
| `/side сообщение` или `/btw сообщение` | Создает изолированную фоновую задачу для поиска информации или расчетов, которая не прерывает основную нить разговора. |
| `/skill имя-навыка сообщение` | Поручает выполнение запроса конкретному установленному навыку (Skill). |
| `/steer сообщение` | Прерывает текущий вызов инструмента моделью и корректирует генерацию ответа новыми инструкциями. |
| `/stop сообщение` | Останавливает выполнение активной задачи и переводит чат в режим ожидания; в сообщении можно указать причину. |
| `/cancel сообщение` | Отменяет текущую сессию диалога; в сообщении можно указать причину. |

Примеры использования:

```text
/goal Закончить подготовку релиза: запустить тесты, проверить changelog, убедиться в правильности скрипта установки и описать риски.
/loop каждые 5 минут /side Проверять очередь деплоя и составлять отчет о блокирующих проблемах.
/side Проверить, несет ли обновление этой зависимости риски совместимости.
/skill git-assistant Написать сообщение коммита на основе текущих изменений diff.
/steer Прекрати рефакторинг и сначала просто исправь ошибку сборки.
```

## Основные команды CLI

| Команда | Назначение |
| --- | --- |
| `anda` | Запуск интерактивного текстового интерфейса в терминале. |
| `anda update` | Обновление программы, установленной с помощью автоматического скрипта. |
| `anda status` | Проверка статуса фонового демона. |
| `anda start` | Запуск фонового демона. |
| `anda stop` | Остановка фонового демона. |
| `anda restart` | Перезапуск фонового демона. |
| `anda models reload` | Перезагрузка списка провайдеров моделей из `config.yaml` без остановки демона. |
| `anda autostart status` | Проверка регистрации службы в списке автозапуска операционной системы. |
| `anda agent run --prompt "..."` | Отправка единичного запроса и ожидание полного ответа без запуска интерфейса терминала. |
| `anda voice --record-secs 8` | Голосовой ввод сообщений (требуется настроенный провайдер транскрипции аудио). |

Для голосового озвучивания ответов также требуется активировать параметр `tts.enabled: true`. Если вам нужен только голосовой ввод с выводом ответов текстом, добавьте параметр `--no-playback`.
