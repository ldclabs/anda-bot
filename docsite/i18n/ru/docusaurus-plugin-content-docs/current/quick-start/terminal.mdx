# Терминал

После запуска `anda` terminal UI подключается к локальному daemon. Успешная беседа отправляется в Brain для формирования долгой памяти.

## Управление вводом

| Клавиша | Действие |
| --- | --- |
| Enter | Отправить сообщение |
| Shift+Enter | Новая строка |
| Ctrl+U | Очистить ввод |
| Ctrl+A / Ctrl+E | В начало или конец |
| Esc | Статус |
| Ctrl+C | Выход |

После изменения `~/.anda/config.yaml`:

```text
/reload
```

## Slash-команды

| Команда | Когда использовать |
| --- | --- |
| `/goal message` | Длинный цикл цели. |
| `/loop message` | Повторяет prompt или slash-команду. Опишите интервал естественным языком или пропустите его, чтобы модель сама выбрала темп. |
| `/side message` или `/btw message` | Боковая задача без прерывания основной беседы. |
| `/skill skill-name message` | Конкретная skill. |
| `/steer message` | Коррекция текущего ответа. |
| `/stop message` | Остановить текущую задачу и оставить беседу в idle. |
| `/cancel message` | Отменить активную сессию беседы. |

```text
/goal Заверши release checklist до подтверждения тестов и changelog.
/loop каждые 5 минут /side Проверь очередь деплоя и перечисли блокеры.
/side Проверь риск для macOS arm64.
/skill git-assistant Напиши commit message по diff.
/steer Не рефактори, сначала почини build.
```

## CLI-команды

| Команда | Назначение |
| --- | --- |
| `anda` | Открыть интерактивный UI. |
| `anda update` | Обновить установку из скрипта. |
| `anda status` | Показать, запущен ли daemon. |
| `anda start` | Запустить daemon. |
| `anda stop` | Остановить daemon. |
| `anda restart` | Перезапустить daemon. |
| `anda autostart status` | Показать, зарегистрирован ли автозапуск при входе. |
| `anda agent run --prompt "..."` | Отправить запрос и дождаться полного результата без UI. |
| `anda voice --record-secs 8` | Запустить голосовой ввод. |

Для воспроизведения голоса нужен `tts.enabled: true`. Для голосового ввода и текстового ответа используйте `--no-playback`.
