# Установить Anda

Anda Bot можно установить в виде скомпилированного официального релиза или запустить напрямую из исходного кода репозитория. Готовые релизы рекомендуются для повседневного использования; запуск из исходников лучше всего подходит для участия в разработке, отладки кода и экспериментов с новыми возможностями.

## Установка последнего релиза

На операционной системе macOS рекомендуется использовать пакетный менеджер Homebrew:

```bash
brew install ldclabs/tap/anda
```

При этом Homebrew установит как консольную утилиту `anda`, так и графический лаунчер `anda_launcher`. Запустите `anda_launcher` один раз для инициализации приложения в строке меню и обновления ярлыка в системной папке `~/Applications/Anda Bot.app`.

Пользователи macOS и Linux также могут воспользоваться скриптом установки:

```bash
curl -fsSL https://raw.githubusercontent.com/ldclabs/anda-bot/main/scripts/install.sh | sh
```

Пользователям Windows необходимо скачать файл `AndaBotSetup-windows-x86_64.exe` со страницы [последних релизов](https://github.com/ldclabs/anda-bot/releases/latest) и запустить его. Установщик поместит Anda в папку `%LOCALAPPDATA%\Programs\AndaBot`, установит встроенные навыки (Skills), создаст ярлыки в меню «Пуск» и на рабочем столе, зарегистрирует лаунчер в системном трее для автозапуска при входе в систему, откроет графический мастер настройки для указания провайдера, API-ключа и используемой модели, а также настроит проверку фоновых обновлений с запросом на перезапуск при их готовности.

Опытные пользователи и скрипты CI могут использовать командную строку PowerShell:

```powershell
irm https://raw.githubusercontent.com/ldclabs/anda-bot/main/scripts/install.ps1 | iex
```

Скрипт установки для macOS также копирует приложение в `~/Applications/Anda Bot.app`, настраивает запуск лаунчера при входе в систему и сразу же его открывает. Лаунчер запустит фонового демона после окончания настройки, сможет проверять обновления и выдаст уведомление с предложением перезапуска при их скачивании. При установке через скрипт на Linux автозапуск фоновой службы регистрируется напрямую в системе. В PowerShell вы можете использовать параметры `-NoAutostart` или `-NoStart` для отмены автозапуска; для bash-скрипта задайте переменные окружения `ANDA_NO_AUTOSTART=1` или `ANDA_NO_START=1`.

После завершения установки откройте новое окно терминала и выполните:

```bash
anda
```

## Подготовка API-ключа модели

Вам потребуется хотя бы один API-ключ от провайдера языковых моделей. Пользователи установщика для Windows могут ввести его прямо в мастере настройки. При запуске через консоль CLI можно временно задать переменную окружения:

```bash
DEEPSEEK_API_KEY=ВАШ_API_KEY anda
```

В консоли Windows PowerShell:

```powershell
$env:DEEPSEEK_API_KEY="ВАШ_API_KEY"; anda
```

Вы также можете сохранить ключ в конфигурационном файле `~/.anda/config.yaml` для постоянного использования. Значение `api_key` из файла `config.yaml` имеет более высокий приоритет по сравнению с переменными окружения.

## Первый запуск

При первом запуске Anda автоматически создает локальную рабочую директорию и шаблон файла `~/.anda/config.yaml`. Если конфигурация окажется неполной, текстовый интерфейс терминала выведет список параметров, требующих заполнения.

1. Откройте файл `~/.anda/config.yaml`.
2. Заполните поля провайдера моделей: семейство (`family`), название модели (`model`), адрес API (`api_base`) и сам ключ (`api_key`) либо оставьте поле ключа пустым при использовании переменных окружения.
3. Сохраните изменения.
4. Обновите конфигурацию из лаунчера, боковой панели браузера или выполнив команду `anda models reload`.

Минимальный пример конфигурации:

```yaml
model:
  active: "deepseek-v4-pro"
  providers:
    - family: anthropic
      model: "deepseek-v4-pro"
      api_base: "https://api.deepseek.com/anthropic"
      api_key: "ВАШ_API_KEY" # Можно оставить пустым при установленной переменной DEEPSEEK_API_KEY
      labels: ["pro", "brain"]
      disabled: false
```

Метка `brain` указывает системе долгосрочной памяти использовать данного провайдера по умолчанию. Если ни у одного из провайдеров нет этой метки, задачи памяти будут выполняться на текущей активной модели (active model).

## Запуск из исходного кода

Для прямого запуска из склонированного репозитория:

```bash
git clone https://github.com/ldclabs/anda-bot.git
cd anda-bot
cargo run -p anda_bot --
```

Вы можете изолировать настройки, проекты и тестовые данные разных учетных записей, переопределив домашний каталог:

```bash
anda --home /путь/к/.anda
```

Эквивалентная команда для запуска из исходников:

```bash
cargo run -p anda_bot -- --home /путь/к/.anda
```

## Обновление

Если Anda была установлена с помощью скрипта автоматической установки, обновите ее командой:

```bash
anda update
```

Для управления фоновой службой (демоном) используйте следующие команды:

```bash
anda status
anda start
anda stop
anda restart
anda autostart status
```
