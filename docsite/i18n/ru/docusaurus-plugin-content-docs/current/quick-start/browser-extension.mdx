# Расширение для браузера

Расширение Anda Bot открывает интерфейс Anda во встроенной боковой панели браузеров на базе Chromium, таких как Google Chrome и Microsoft Edge. Оно выступает в роли связующего звена с локальным демоном Anda, запущенным на вашем компьютере, благодаря чему контекст браузера, терминальные инструменты, файлы проектов, навыки (Skills), субагенты, каналы коммуникации и общая память Brain работают в рамках единой локальной среды.

## Требования

- Браузер Google Chrome или Microsoft Edge версии 116 или новее (либо любой другой браузер на базе Chromium).
- Установленное приложение Anda, настроенное как минимум с одним API-ключом провайдера моделей.
- Локальная служба (daemon) Anda, запущенная на вашем компьютере.

Установите расширение из магазина дополнений вашего браузера:

- [Anda Bot в Chrome Web Store](https://chromewebstore.google.com/detail/anda-bot/injpfajmddchcphfkdkiflfddmajglfd) — для Google Chrome и других Chromium-совместимых браузеров.
- Microsoft Edge: откройте страницу **Дополнения Edge** (`edge://extensions` → «Получить расширения для Microsoft Edge») и найдите **Anda Bot**.

Процесс установки и настройки идентичен для обоих браузеров.

## Подключение боковой панели

Запустите локального демона:

```bash
anda start
```

Сгенерируйте токен авторизации браузера:

```bash
anda browser token --days 365
```

Скопируйте выведенные Gateway URL и Bearer-токен. Откройте боковую панель Anda Bot in Chrome или Edge, перейдите в настройки, вставьте скопированные значения и сохраните их. После успешного подключения вы сможете общаться с локальным агентом прямо с любой открытой веб-страницы.

## Работа с контекстом страницы

При отправке сообщения из боковой панели Anda автоматически получает заголовок и URL-адрес текущей активной вкладки в качестве контекста запроса. Если задача требует взаимодействия с веб-страницей, агент может использовать браузерные инструменты для инспектирования контента и управления вкладками.

Примеры использования:

- Быстро составить резюме или сравнить содержимое читаемых веб-страниц.
- Собрать информацию из нескольких вкладок с сохранением ссылок на первоисточники.
- Автоматически заполнять формы, кликать по кнопкам, прокручивать страницы, делать скриншоты или экспортировать страницы в PDF, если этого требует задача.
- Подготавливать консольные команды или анализировать локальные файлы проекта с сохранением контекста открытой веб-страницы.
- Добавлять важные ответы ассистента в закладки, организовывать их по папкам и быстро возвращаться к исходным обсуждениям.

## Возможности управления браузером

Расширение предоставляет локальному демону широкий набор браузерных инструментов. К ним относятся:

- Вывод списка вкладок, переключение между ними, открытие, закрытие, перезагрузка страниц и навигация.
- Чтение выделенного текста, извлечение всего текста страницы, создание скриншотов, анализ структуры доступности (accessibility data) и экспорт страниц в PDF.
- Выполнение кликов, ввод текста, эмуляция нажатий клавиш, прокрутка, выбор опций в выпадающих списках, загрузка файлов и работа с модальными диалогами.
- Скачивание файлов и инспектирование загрузок, инициированных браузером.

Все действия выполняются в контексте активной в данный момент вкладки. Если для выполнения задачи необходимо переключиться на другую вкладку, попросите Anda сделать это.

Чтобы разрешить открытие локальных файлов или папок с помощью инструмента `open_file`, включите опцию `Разрешить открывать локальные файлы по ссылкам` (Allow access to file URLs) в настройках расширения Anda Bot в вашем браузере (в Edge: `edge://extensions` → Anda Bot → Подробнее/Details).

## Локальная среда и конфиденциальность

Расширение сохраняет Gateway URL, Bearer-токен, ID сессии браузера и другие параметры в локальном хранилище браузера. Вся информация (промпты, выбранные файлы, скриншоты, текст веб-страниц) передается напрямую на запущенный вами локальный демон Anda.

Обработка этих данных происходит локально на вашем компьютере и на стороне настроенных вами провайдеров моделей. Используйте расширение только с локальным демоном и моделями, которым вы доверяете, особенно при работе со страницами, содержащими конфиденциальные данные.

## Внешние каналы связи

Боковая панель браузера — это лишь один из интерфейсов. Если вы настроите интеграцию с WeChat, Telegram, Discord или Lark / Feishu, сообщения из этих каналов будут поступать на тот же локальный демон, запущенный на вашем компьютере.

Подробности настройки каналов описаны в разделе [Каналы связи и голос](../runtime/channels).

## Устранение неполадок

- Если боковая панель не подключается, убедитесь, что служба `anda start` запущена, а Gateway URL и Bearer-токен скопированы без ошибок.
- Если браузерные действия завершаются ошибкой на системных страницах вроде `chrome://`, `edge://` или в магазине расширений, перейдите на обычный веб-сайт. Браузеры на базе Chromium из соображений безопасности блокируют работу скриптов расширений на служебных страницах.
- При появлении ошибок моделей проверьте правильность указания API-ключей в файле `~/.anda/config.yaml` или в переменных окружения, используемых при запуске службы.
