# Долгосрочные задачи и субагенты

Anda Bot спроектирован для достижения сложных, многоэтапных целей, требующих непрерывности контекста, а не просто ответов на одиночные вопросы. В ходе выполнения долгосрочной задачи Anda отслеживает прогресс, сжимает контекст при его разрастании и продолжает выполнение шагов до достижения требуемого результата.

## Использование /goal

Команда `/goal` помечает ваш запрос как долгосрочную цель:

```text
/goal Закончить подготовку релиза: запустить тесты, проверить changelog, убедиться в правильности скрипта установки и описать риски.
```

В моменты простоя основного агента система контроля целей (goal checker) оценивает, является ли полученный результат полностью готовым. Если требуются дополнительные действия, она формирует новое поручение для основного агента. При чрезмерном разрастании контекста Anda сжимает текущее состояние сессии и открывает связанный дочерний диалог для продолжения работы.

## Использование /loop

Команда `/loop` запускает периодическое циклическое выполнение промпта или команды со слэшем. Вы можете указать временной интервал в свободной форме на естественном языке, например `every 5 minutes` (каждые 5 минут), `5m` или используя русские обозначения времени. Модель распознает интервал и при необходимости создаст периодическую задачу cron:

```text
/loop каждые 5 минут /side Проверять очередь деплоя и составлять отчет о блокирующих проблемах.
```

Если интервал времени не указан, Anda запустит цикл с автоматическим определением темпа на стороне модели, которая будет продолжать выполнение шагов по мере готовности результатов:

```text
/loop Продолжать проверку миграции до тех пор, пока каждый шаг валидации не подтвердит успешное выполнение.
```

## Использование /side

Команда `/side` (или альтернативная `/btw`) переносит выполнение запроса в изолированную фоновую задачу. Это крайне полезно для:

- Исследования потенциальных рисков.
- Запроса альтернативного мнения (review) у модели касательно текущих выводов.
- Сбора предыстории и справочных данных без прерывания основной линии диалога.

```text
/side Проверить, несет ли обновление этой зависимости риски для пользователей macOS arm64.
```

## Использование /skill

Пользовательские навыки работы (Personal runtime skills) сохраняются в каталоге `~/.anda/skills`, а встроенные системные навыки загружаются из `~/.anda/bundled-skills`. Если вам требуется запустить специфический готовый сценарий для обработки запроса, используйте:

```text
/skill git-assistant Написать сообщение коммита по стандарту Conventional Commit на основе текущих изменений diff.
```

## Роль субагентов

Субагенты могут брать на себя специализированные роли, такие как исследователь (research), разработчик (implementation), рецензент (review) и контролер (supervision). Основная сессия диалога всегда хранит главную цель и выносит финальный вердикт о качестве работы, тогда как субагенты сфокусированно выполняют порученные им локальные задачи.

Подобная архитектура эффективна для:

- Исследовательских задач, требующих чтения большого количества файлов документации.
- Программирования, где процессы написания кода и его проверки должны быть разделены.
- Целей, требующих многократных независимых проверок фактического выполнения критериев готовности.
- Работ, требующих подключения внешних утилит без потери контекста основного разговора.

## Инструменты и рабочая область

В базовый набор инструментов Anda входят работа с памятью, ведение заметок, выполнение консольных команд shell, чтение/запись файлов, списки дел (todo), навыки, планировщик cron и создание субагентов. Инструменты работы с файлами и консолью по умолчанию выполняются в каталоге `~/.anda/workspace`.

При поручении задач, связанных с кодом текущего репозитория или внешними утилитами, всегда четко формулируйте цель, критерии приемки и ограничения.

```text
/goal Исправить сборку документации в текущем репозитории. Не изменять код Rust. Готово означает, что команда pnpm --dir docsite build завершается успешно.
```

## Когда не нужно использовать /goal или /loop

Для простых вопросов и ответов, разовых объяснений или поиска справочных команд в терминале использовать `/goal` или `/loop` не требуется — пишите обычные сообщения. Сохраняйте `/goal` для задач, требующих жесткого аудита критериев готовности, а `/loop` — для периодического или автоматического фонового мониторинга.
