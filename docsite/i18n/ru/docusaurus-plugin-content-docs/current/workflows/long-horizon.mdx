# Длинные задачи и сабагенты

Anda Bot создан для сложных целей, которым нужна непрерывность. Он хранит цель, проверяет прогресс, сжимает контекст и продолжает при необходимости.

## Использовать /goal

```text
/goal Заверши release: тесты, changelog, docs установки и резюме рисков.
```

`/goal` или `/loop` открывает длинный цикл. Когда главный агент свободен, проверяющий решает, завершено ли дело или надо продолжать.

## Использовать /side

`/side` или `/btw` создает боковую задачу для исследования, проверки или сбора контекста без остановки основной беседы.

```text
/side Проверь, влияет ли эта зависимость на macOS arm64.
```

## Использовать /skill

```text
/skill git-assistant Напиши Conventional Commit по текущему diff.
```

## Роль сабагентов

Они могут исследовать, реализовывать, проверять или контролировать. Главная сессия хранит цель и финальное решение; сабагенты выполняют фокусную работу.

## Инструменты и workspace

Anda включает память, заметки, shell, файлы, tasks, skills, cron и сабагентов. Shell и файлы работают в `~/.anda/workspace` по умолчанию.

Указывайте цель, критерии приемки и границы.

```text
/goal Исправь docs build. Rust не меняй. Готово, когда pnpm --dir docsite build проходит.
```
