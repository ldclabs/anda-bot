# Каналы и голос

Anda может оставаться в терминале или подключаться к командным каналам. Каналы возвращают контекст в тот же runtime и память.

## Поддерживаемые каналы

- Telegram
- WeChat
- Discord
- Lark / Feishu

Настройка находится в `channels` внутри `~/.anda/config.yaml`.

## Несколько пользователей

Один daemon может обслуживать нескольких доверенных пользователей через того же агента Anda. Создайте пользовательский ключ, затем укажите этот id в поле `user` канала. Если `user` не задан, канал использует идентичность локального владельца, сохраненную в защищенном хранилище учетных данных ОС.

```bash
anda user create alice
anda user list
```

Команда записывает новый публичный ключ в `users` и сохраняет соответствующий приватный ключ в защищенном хранилище учетных данных ОС, например Apple Keychain, Windows Credential Manager или Linux Secret Service. Старые ключи owner и daemon импортируются в защищенное хранилище при первом использовании; передавайте `--key-path` только если явно нужен файловый ключ.

В Linux, если поставщик Secret Service недоступен или не разблокирован, Anda переключается на файлы приватных ключей в `~/.anda/keys/` и выводит/записывает предупреждение. Чтобы использовать Secret Service, запустите и разблокируйте поставщика в пользовательской D-Bus-сессии, например `gnome-keyring-daemon --start --components=secrets`, убедитесь, что `DBUS_SESSION_BUS_ADDRESS` задан для процесса Anda, затем перезапустите Anda. Пользователи KDE могут разблокировать KWallet.

Чтобы экспортировать существующий приватный ключ идентичности в файл, используйте `anda user export`. Идентичность может быть `daemon`, `owner`, `default` или id доверенного пользователя:

```bash
anda user export daemon --key-path ./anda-daemon.key
anda user export owner --key-path ./anda-owner.key
anda user export alice --key-path ./alice.key
```

Не помещайте экспортированные файлы приватных ключей в репозитории исходного кода или общие папки.

Конфигурация остается такой:

```yaml
users:
  - id: alice
    pubkey: "ALICE_ED25519_PUBLIC_KEY"
```

## Telegram

```yaml
channels:
  telegram:
    - id: personal
      user: alice
      bot_token: "YOUR_TELEGRAM_BOT_TOKEN"
      username: "YOUR_TELEGRAM_BOT_USERNAME"
      allowed_users:
        - "*"
      mention_only: false
```

## WeChat

```yaml
channels:
  wechat:
    - id: personal
      bot_token: ""
      username: anda-wechat
      allowed_users:
        - "*"
```

Если `bot_token` пустой, используйте `anda channel init wechat` и QR-вход.

## Discord

Discord использует bot token и Gateway-соединение. Сначала создайте и установите bot, затем меняйте `config.yaml`:

1. Откройте [Discord Developer Portal](https://discord.com/developers/applications), затем создайте или откройте app.
2. На странице **Bot** скопируйте или сбросьте bot token. Также включите **Message Content Intent** в **Privileged Gateway Intents**, чтобы Anda могла читать обычный текст сообщений.
3. На странице **Installation** убедитесь, что **Guild Install** содержит scope `bot`. Без `bot` app может быть видна в Discord Apps, но не появится как bot-участник сервера.
4. Выдайте минимум `View Channels`, `Send Messages` и `Read Message History`. Добавьте `Attach Files` для ответов с файлами и `Add Reactions`, если `ack_reactions: true`.
5. Установите app на сервер. После серверной установки bot должен появиться в списке участников.

```yaml
channels:
  discord:
    - id: server
      bot_token: "YOUR_DISCORD_BOT_TOKEN"
      username: anda-discord
      guild_id: "YOUR_DISCORD_GUILD_ID"
      allowed_users:
        - "YOUR_DISCORD_USER_ID"
      mention_only: true
      ack_reactions: true
```

Для проверки конфигурации обязателен только `bot_token`. Для рабочей и более безопасной настройки задайте `allowed_users` как Discord user IDs; при пустом `allowed_users` и `allow_external_users: false` Anda игнорирует неразрешённых отправителей.

```bash
anda channel list
anda restart
```

В канале сервера упомяните bot, например `@anda-discord суммируй этот тред`. В DM с bot отправьте обычное сообщение без упоминания.

## Lark / Feishu

```yaml
channels:
  lark:
    - id: work
      app_id: "cli_xxx"
      app_secret: "YOUR_APP_SECRET"
      platform: feishu
      receive_mode: websocket
      mention_only: true
```

## Безопасность

- `allowed_users` определяет, кто может активировать Anda.
- Используйте `"*"` только если открытый доступ допустим.
- Lark/Feishu требует `app_id` и `app_secret`.
- Роуты каналов сохраняются, чтобы отвечать в правильный поток или комнату.

## Голос

```bash
anda voice --record-secs 8
```

Голосовой ввод требует `transcription.enabled: true`; озвучивание требует `tts.enabled: true`. Для голоса на входе и текста на выходе:

```bash
anda voice --record-secs 8 --no-playback
```
