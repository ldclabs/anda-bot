# تكوين بيئة التشغيل

يقوم Anda بتخزين حالة بيئة التشغيل في `~/.anda` افتراضياً. ملف التكوين الرئيسي هو `~/.anda/config.yaml` ، ويتم إنشاؤه تلقائياً عند التشغيل الأول. يقع تكوين خادم MCP الاختياري في `~/.anda/mcp.json`.

## مزودو النماذج

يجب أن يتضمن المزود النشط (active) العائلة (`family`)، والنموذج (`model`)، ورابط قاعدة API (`api_base`). يمكن كتابة مفتاح `api_key` في ملف التكوين أو تركه فارغاً وتقديمه عبر متغير بيئة.

```yaml
model:
  active: "deepseek-v4-pro"
  providers:
    - family: anthropic
      model: "deepseek-v4-pro"
      api_base: "https://api.deepseek.com/anthropic"
      api_key: "YOUR_API_KEY"
      labels: ["pro", "brain"]
      disabled: false
```

متغيرات بيئة مفتاح API للنموذج المدعومة هي:

```text
OPENAI_API_KEY
ANTHROPIC_API_KEY
GEMINI_API_KEY
GOOGLE_API_KEY
DEEPSEEK_API_KEY
MINIMAX_API_KEY
MIMO_API_KEY
MOONSHOT_API_KEY
KIMI_API_KEY
BIGMODEL_API_KEY
GLM_API_KEY
```

## الإعدادات العامة للمستوى الأعلى

| الإعداد | الغرض |
| --- | --- |
| `addr` | عنوان البوابة المحلية، الافتراضي هو `127.0.0.1:8042`. |
| `sandbox` | إذا تم تمكينه (true)، فسيتم تشغيل أوامر shell تحت `~/.anda/sandbox`. |
| `https_proxy` | البروكسي المستخدم لطلبات النموذج والقنوات. |
| `model` | مزودو النماذج للوكيل ودماغ الذاكرة (Brain). |
| `transcription` | مزود خدمة الترانسكريبت للمدخلات الصوتية والملفات الصوتية المرفقة. |
| `tts` | مزود خدمة توليد الصوت (TTS) للمخرجات الصوتية. |
| `channels` | تكوينات Telegram و WeChat و Discord و Lark/Feishu والقنوات الأخرى. |

## خوادم MCP

يمكن لـ Anda Bot الاتصال بخوادم MCP عبر عمليات stdio الفرعية أو نقاط نهاية HTTP. ضع تكوين MCP المحمول في `~/.anda/mcp.json` ، ثم أعد تشغيل daemon. يقبل الملف كلاً من المفاتيح الجذرية `mcpServers` و `servers` لمساعدتك في لصق التكوينات من الأدوات المتوافقة الأخرى مع MCP مع الحد الأدنى من التعديلات.

```json
{
  "mcpServers": {
    "filesystem": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "$ANDA_WORKSPACE"]
    },
    "remote": {
      "type": "http",
      "url": "https://mcp.example.com/mcp",
      "headers": {
        "Authorization": "Bearer ${MCP_REMOTE_TOKEN}"
      }
    }
  }
}
```

تظهر أدوات MCP البعيدة كأدوات Anda ملقبة بـ `mcp_<server_id>_<remote_tool>`. تدعم سلاسل التكوين توسيع متغيرات البيئة `$VAR` و `${VAR}`. كما أن `ANDA_HOME` و `ANDA_WORKSPACE` مدمجتان بشكل افتراضي. بالنسبة لخوادم stdio، يتم تشغيل الأمر `command` مباشرة بدون shell؛ وعند حذف `cwd` ، يتم استخدام مساحة عمل Anda الأولى. وإذا تم حذف `type` ، يقوم Anda باستنتاج `stdio` من معلمة `command` أو `http` من عنوان URL.

يمكن للوكيل أيضاً إضافة خادم إلى بيئة التشغيل باستخدام `add_mcp_server`. قم بتعيين `persist: false` للـ daemon الحالي فقط، أو `persist: true` لحفظ الخادم في `~/.anda/mcp.json` لإعادة التشغيل في المستقبل. تتطابق حقول الخادم مع إدخال `mcp.json` : `type` و `command` و `args` و `env` و `cwd` و `url` و `headers` و `enabled` و `include` و `exclude` ، بالإضافة إلى الحقول الخاصة بالأدوات `id` و `persist`.

## هيكل المجلد المحلي

```text
~/.anda/
  config.yaml
  mcp.json
  anda-daemon.pid
  channels/
  db/
  keys/ # المفاتيح الموروثة أو المصدرة صراحة أو المفاتيح الاحتياطية لـ Secret Service
  logs/
  bundled-skills/
  sandbox/
  skills/
  skills-manifest.json
  skill-backups/
  skill-trash/
  workspace/
```

| الدليل | محتويات الدليل |
| --- | --- |
| `db/` | قاعدة الذاكرة المحلية والمحادثات وحالة القنوات والمهام المجدولة (cron) والكائنات والبيانات ذات الصلة. |
| `keys/` | ملفات مفاتيح التوقيع القديمة أو المصدرة صراحة أو المفاتيح الاحتياطية لنظام Linux Secret Service. يتم تخزين المفاتيح الخاصة بالهوية الجديدة في مخزن أوراق الاعتماد الآمن لنظام التشغيل بشكل افتراضي عندما يكون متاحاً. |
| `logs/` | ملفات سجلات daemon و CLI. |
| `channels/` | ملفات حالة تشغيل القنوات. |
| `bundled-skills/` | المهارات المنسقة المدارة بالإصدارات والمثبتة مع تحديثات Anda. |
| `sandbox/` | دليل عزل shell عند تمكين وضع الحماية (sandbox). |
| `skills/` | المهارات الشخصية لبيئة التشغيل التي تم إنشاؤها أو استنساخها أو تعديلها من قبل المستخدم. |
| `skills-manifest.json` | بيانات الحالة (التمكين/التعطيل) والتحديث لمكتبة المهارات المحلية. |
| `skill-backups/` | نسخ احتياطية للمهارات مع طوابع زمنية قبل تعديلها في لوحة التحكم. |
| `skill-trash/` | سلة مهملات المهارات الشخصية المحذوفة قبل إزالتها نهائياً. |
| `workspace/` | مساحة العمل الافتراضية لأدوات الملفات والـ shell. |

## حدود الخصوصية

يتم تخزين بيانات الحالة المحلية لـ Anda، ومخطط الذاكرة، والمحادثات، وحالة القنوات، وحالة المهام المجدولة، وسجلات التشغيل، وبيانات مساحة العمل افتراضياً على جهازك الخاص. وتُخزن المفاتيح الخاصة بالهوية افتراضياً في مخزن أوراق الاعتماد الآمن لنظام التشغيل، بينما يتم الاحتفاظ بملفات المفاتيح الصريحة ومفاتيح Linux Secret Service الاحتياطية في دليل `~/.anda/keys/`.

على نظام Linux، إذا لم يكن هناك مزود لخدمة Secret Service متاحاً أو تم إلغاء قفله، فسيرجع Anda إلى استخدام ملفات المفاتيح الخاصة تحت المسار `~/.anda/keys/` ويطبع/يسجل تحذيراً. لاستخدام Secret Service، ابدأ تشغيل المزود وافتح قفله في جلسة D-Bus للمستخدم (على سبيل المثال، `gnome-keyring-daemon --start --components=secrets`) ، وتأكد من تعيين متغير البيئة `DBUS_SESSION_BUS_ADDRESS` لعملية Anda، ثم أعد تشغيل Anda. يمكن لمستخدمي بيئة KDE فتح قفل KWallet بدلاً من ذلك.

ومع ذلك، قد لا يزال مزودو النماذج يتلقون الأسئلة وملخصات نتائج الأدوات وطلبات تكوين أو استدعاء الذاكرة. يرجى اختيار مزودين موثوقين، ونقاط نهاية خاصة، وسياسات بروكسي بناءً على متطلبات الخصوصية الخاصة بك.

:::tip
استخدم المعلمة `anda --home /path/to/.anda` عندما تحتاج إلى بيانات معزولة لهويات مختلفة، أو مشاريع مختلفة، أو لاختبارات معينة.
:::
