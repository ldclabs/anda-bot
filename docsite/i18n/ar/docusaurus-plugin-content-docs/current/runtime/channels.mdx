# القنوات والصوت

يمكن أن يبقى Anda في الطرفية أو ينضم إلى قنوات الفريق. تعيد القنوات السياق إلى وقت التشغيل ونظام الذاكرة نفسه.

## القنوات المدعومة

- Telegram
- WeChat
- Discord
- Lark / Feishu

يوجد الإعداد تحت `channels` في `~/.anda/config.yaml`.

## المستخدمون المتعددون

يمكن لـ daemon واحد خدمة عدة مستخدمين موثوقين عبر وكيل Anda نفسه. أنشئ مفتاح مستخدم ثم استخدم ذلك id في حقل `user` داخل القناة. عند حذف `user` تستخدم القناة هوية المالك المحلي المخزنة في مخزن بيانات الاعتماد الآمن لنظام التشغيل.

```bash
anda user create alice
anda user list
```

يكتب الأمر المفتاح العام الجديد تحت `users` ويحفظ المفتاح الخاص المطابق في مخزن بيانات الاعتماد الآمن لنظام التشغيل، مثل Apple Keychain أو Windows Credential Manager أو Linux Secret Service. تستورد مفاتيح owner و daemon القديمة إلى المخزن الآمن عند أول استخدام. استخدم `anda user export` أدناه عندما تحتاج صراحة إلى مفتاح ملف.

على Linux، إذا لم يكن مزود Secret Service متاحا أو مفتوحا، ينتقل Anda إلى ملفات مفاتيح خاصة تحت `~/.anda/keys/` ويعرض/يسجل تحذيرا. لاستخدام Secret Service، شغل وافتح مزودا داخل جلسة D-Bus للمستخدم، مثلا `gnome-keyring-daemon --start --components=secrets`، وتأكد من ضبط `DBUS_SESSION_BUS_ADDRESS` لعملية Anda، ثم أعد تشغيل Anda. يمكن لمستخدمي KDE فتح KWallet بدلا من ذلك.

لتصدير مفتاح هوية خاص موجود إلى ملف، استخدم `anda user export`. يمكن أن تكون الهوية `daemon` أو `owner` أو `default` أو id مستخدم موثوق:

```bash
anda user export daemon --key-path ./anda-daemon.key
anda user export owner --key-path ./anda-owner.key
anda user export alice --key-path ./alice.key
```

لا تضع ملفات المفاتيح الخاصة المصدرة في مستودعات الكود أو المجلدات المشتركة.

يبقى شكل الإعدادات هكذا:

```yaml
users:
  - id: alice
    pubkey: "ALICE_ED25519_PUBLIC_KEY"
```

## Telegram

```yaml
channels:
  telegram:
    - id: personal
      user: alice
      bot_token: "YOUR_TELEGRAM_BOT_TOKEN"
      username: "YOUR_TELEGRAM_BOT_USERNAME"
      allowed_users:
        - "*"
      mention_only: false
```

## WeChat

```yaml
channels:
  wechat:
    - id: personal
      bot_token: ""
      username: anda-wechat
      allowed_users:
        - "*"
```

عند ترك `bot_token` فارغًا يمكن استخدام `anda channel init wechat` ومسح QR.

## Discord

يستخدم Discord رمز bot واتصال Gateway. أنشئ وثبّت bot قبل تعديل `config.yaml`:

1. افتح [Discord Developer Portal](https://discord.com/developers/applications)، ثم أنشئ app أو افتح app موجودة.
2. في صفحة **Bot** انسخ رمز bot أو أعد إنشاءه. فعّل أيضًا **Message Content Intent** ضمن **Privileged Gateway Intents** حتى يستطيع Anda قراءة نص الرسائل العادية.
3. في **Installation** تأكد أن **Guild Install** يحتوي على scope باسم `bot`. من دون `bot` قد تظهر app في Discord Apps لكنها لن تظهر كعضو bot في الخادم.
4. امنح على الأقل `View Channels` و `Send Messages` و `Read Message History`. أضف `Attach Files` لردود الملفات، وأضف `Add Reactions` إذا كان `ack_reactions: true`.
5. ثبّت app في الخادم. بعد تثبيت الخادم يجب أن يظهر bot في قائمة الأعضاء.

```yaml
channels:
  discord:
    - id: server
      bot_token: "YOUR_DISCORD_BOT_TOKEN"
      username: anda-discord
      guild_id: "YOUR_DISCORD_GUILD_ID"
      allowed_users:
        - "YOUR_DISCORD_USER_ID"
      mention_only: true
      ack_reactions: true
```

الحقل الإلزامي الوحيد في التحقق من الإعداد هو `bot_token`. لإعداد عملي وأكثر أمانًا، اضبط `allowed_users` على Discord user IDs؛ عند ترك `allowed_users` فارغًا مع `allow_external_users: false` يتجاهل Anda المرسلين غير المسموح لهم.

```bash
anda channel list
anda restart
```

في قناة الخادم اذكر bot مثل `@anda-discord لخص هذا النقاش`. في DM مع bot أرسل رسالة عادية من دون ذكره.

## Lark / Feishu

```yaml
channels:
  lark:
    - id: work
      app_id: "cli_xxx"
      app_secret: "YOUR_APP_SECRET"
      platform: feishu
      receive_mode: websocket
      mention_only: true
```

## الأمان

- يحدد `allowed_users` من يمكنه تشغيل Anda.
- استخدم `"*"` فقط عند قبول الوصول المفتوح.
- يحتاج Lark/Feishu إلى `app_id` و `app_secret`.
- تحفظ مسارات القنوات للرد في الخيط أو الغرفة الصحيحة.

## الصوت

```bash
anda voice --record-secs 8
```

يتطلب الإدخال الصوتي `transcription.enabled: true` ويتطلب تشغيل الرد `tts.enabled: true`. للصوت كإدخال والنص كمخرج:

```bash
anda voice --record-secs 8 --no-playback
```
