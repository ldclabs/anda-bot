# تثبيت Anda

يمكن تثبيت Anda Bot من إصدار رسمي (release) أو تشغيله مباشرة من مصدر المستودع. تعد الإصدارات الرسمية هي الأفضل للاستخدام اليومي؛ بينما تعد عمليات البناء من المصدر هي الأفضل للمساهمة والتصحيح وتجربة الميزات الجديدة.

## تثبيت الإصدار الأخير

على نظام macOS، يوصى باستخدام Homebrew:

```bash
brew install ldclabs/tap/anda
```

على نظام macOS، تقوم صيغة Homebrew بتثبيت كل من `anda` و `anda_launcher`. قم بتشغيل `anda_launcher` مرة واحدة لبدء تشغيل مشغل شريط القوائم وتحديث `~/Applications/Anda Bot.app`.

يمكن لمستخدمي macOS و Linux أيضاً استخدام سكريبت التثبيت:

```bash
curl -fsSL https://raw.githubusercontent.com/ldclabs/anda-bot/main/scripts/install.sh | sh
```

يجب على مستخدمي نظام Windows تنزيل `AndaBotSetup-windows-x86_64.exe` من [أحدث إصدار](https://github.com/ldclabs/anda-bot/releases/latest) والنقر عليه نقراً مزدوجاً. يضع برنامج التثبيت Anda تحت المسار `%LOCALAPPDATA%\Programs\AndaBot` ، ويقوم بتثبيت المهارات (Skills) المنسقة، ويقوم بإنشاء اختصارات في قائمة ابدأ وسطح المكتب، ويسجل مشغل علبة النظام للتشغيل عند تسجيل الدخول، ويبدأ تشغيل المشغل، ويفتح معالج إعداد رسومي (GUI) لتكوين المزود/مفتاح API/النموذج، ويفحص التحديثات التي تم تنزيلها والتي يمكن تثبيتها عند إشعار إعادة التشغيل.

لا يزال بإمكان المستخدمين المتقدمين والبيئات المجدولة (CI) استخدام PowerShell:

```powershell
irm https://raw.githubusercontent.com/ldclabs/anda-bot/main/scripts/install.ps1 | iex
```

يقوم مثبت macOS shell أيضاً بتثبيت `~/Applications/Anda Bot.app` ، ويسجل مشغل شريط القوائم عند تسجيل الدخول، ويبدأ تشغيله على الفور; يبدأ المشغل daemon بعد الإعداد، ويمكنه التحقق من وجود تحديثات من شريط القوائم، ويطلب التثبيت وإعادة التشغيل بعد تنزيل التحديث. تسجل عمليات تثبيت Linux shell تشغيل daemon التلقائي مباشرة. بالنسبة إلى PowerShell، أضف `-NoAutostart` أو `-NoStart` لإلغاء التشغيل التلقائي؛ بالنسبة لمثبت shell، اضبط متغيرات البيئة `ANDA_NO_AUTOSTART=1` أو `ANDA_NO_START=1`.

بعد التثبيت، افتح طرفية جديدة وقم بتشغيل:

```bash
anda
```

## إعداد مفتاح API للنموذج

تحتاج إلى مفتاح API واحد على الأقل لمزود النموذج. يمكن لمستخدمي مثبت Windows إدخاله في معالج الإعداد. بينما يمكن لمستخدمي واجهة سطر الأوامر (CLI) البدء بمتغير بيئة:

```bash
DEEPSEEK_API_KEY=YOUR_API_KEY anda
```

في Windows PowerShell:

```powershell
$env:DEEPSEEK_API_KEY="YOUR_API_KEY"; anda
```

يمكنك أيضاً كتابة المفتاح في `~/.anda/config.yaml` لإعادة استخدامه. المفتاح `api_key` في `config.yaml` له الأولوية على متغيرات البيئة.

## التشغيل الأول

عند التشغيل الأول، يقوم Anda بإنشاء دليل تشغيل محلي وملف `~/.anda/config.yaml`. إذا كان التكوين غير مكتمل، تسرد واجهة مستخدم الطرفية الحقول المفقودة.

1. افتح ملف `~/.anda/config.yaml`.
2. املأ حقول مزود النموذج: العائلة (`family`)، والنموذج (`model`)، ورابط قاعدة API (`api_base`)، وإما مفتاح API (`api_key`) أو متغير البيئة.
3. احفظ الملف.
4. قم بتحديث النماذج من المشغل أو اللوحة الجانبية للمتصفح، أو قم بتشغيل `anda models reload`.

تكوين النموذج الأدنى:

```yaml
model:
  active: "deepseek-v4-pro"
  providers:
    - family: anthropic
      model: "deepseek-v4-pro"
      api_base: "https://api.deepseek.com/anthropic"
      api_key: "YOUR_API_KEY" # يمكن تركه فارغاً عند تعيين DEEPSEEK_API_KEY
      labels: ["pro", "brain"]
      disabled: false
```

تجعل تسمية `brain` دماغ الذاكرة يفضل هذا المزود. إذا لم يكن لدى أي مزود هذه التسمية، فستستخدم مهام الذاكرة النموذج النشط (active).

## التشغيل من المصدر

للشغيل مباشرة من المستودع:

```bash
git clone https://github.com/ldclabs/anda-bot.git
cd anda-bot
cargo run -p anda_bot --
```

استخدم دليلاً رئيسياً (home) منفصلاً لعزل الهويات أو المشاريع أو بيانات الاختبار:

```bash
anda --home /path/to/.anda
```

أمر المصدر المعادل:

```bash
cargo run -p anda_bot -- --home /path/to/.anda
```

## التحديث

إذا تم تثبيت التطبيق باستخدام سكريبت التثبيت، فقم بالتحديث باستخدام:

```bash
anda update
```

قم بإدارة daemon الخلفية باستخدام:

```bash
anda status
anda start
anda stop
anda restart
anda autostart status
```
