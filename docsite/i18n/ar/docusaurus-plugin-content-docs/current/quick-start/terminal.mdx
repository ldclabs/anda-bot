# الطرفية

بعد تشغيل `anda` تتصل واجهة الطرفية بالـ daemon المحلي. عند نجاح المحادثة، ترسل في الخلفية إلى Brain لتكوين ذاكرة طويلة.

## التحكم في الإدخال

| المفتاح | الإجراء |
| --- | --- |
| Enter | إرسال الرسالة الحالية |
| Shift+Enter | سطر جديد |
| Ctrl+U | مسح الإدخال |
| Ctrl+A / Ctrl+E | الانتقال للبداية أو النهاية |
| Esc | عرض الحالة |
| Ctrl+C | الخروج |

بعد تعديل `~/.anda/config.yaml`:

```text
/reload
```

## أوامر Slash

| الأمر | الاستخدام |
| --- | --- |
| `/goal message` | وضع هدف في حلقة طويلة. |
| `/loop message` | تشغيل prompt أو أمر slash بشكل متكرر. صف interval باللغة الطبيعية أو احذفه ليحدد النموذج الإيقاع. |
| `/side message` أو `/btw message` | مهمة جانبية لا تقاطع المحادثة الرئيسية. |
| `/skill skill-name message` | استخدام مهارة محددة. |
| `/steer message` | تصحيح الرد الحالي. |
| `/stop message` | إيقاف المهمة الحالية وترك المحادثة في وضع الخمول. |
| `/cancel message` | إلغاء جلسة المحادثة النشطة. |

```text
/goal أكمل قائمة الإصدار حتى تمر الاختبارات و changelog.
/loop كل 5 دقائق /side افحص طابور النشر ولخص العوائق.
/side تحقق من تأثير الاعتماد على macOS arm64.
/skill git-assistant اكتب رسالة commit من diff الحالي.
/steer لا تقم بإعادة هيكلة؛ أصلح build أولًا.
```

## أوامر CLI

| الأمر | الغرض |
| --- | --- |
| `anda` | فتح الواجهة التفاعلية. |
| `anda update` | تحديث تثبيت السكربت. |
| `anda status` | عرض ما إذا كان daemon يعمل. |
| `anda start` | بدء daemon. |
| `anda stop` | إيقاف daemon. |
| `anda restart` | إعادة تشغيل daemon. |
| `anda autostart status` | عرض ما إذا كان التشغيل عند تسجيل الدخول مسجلاً. |
| `anda agent run --prompt "..."` | إرسال طلب وانتظار النتيجة الكاملة دون فتح الواجهة. |
| `anda voice --record-secs 8` | بدء إدخال صوتي. |

يتطلب تشغيل الصوت `tts.enabled: true`. للصوت كإدخال والنص كمخرج استخدم `--no-playback`.
