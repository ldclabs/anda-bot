# الطرفية

بعد تشغيل `anda` ، تتصل واجهة مستخدم الطرفية بـ daemon المحلي. عندما تكتمل المحادثة بنجاح، يتم إرسالها إلى Brain في الخلفية لتكوين ذاكرة طويلة المدى.

## عناصر التحكم في الإدخال

| المفتاح | الإجراء |
| --- | --- |
| Enter | إرسال الرسالة الحالية |
| Shift+Enter | إدخال سطر جديد |
| Ctrl+U | مسح صندوق الإدخال |
| Ctrl+A / Ctrl+E | الانتقال إلى بداية أو نهاية الإدخال |
| Esc | عرض الحالة |
| Ctrl+C | خروج |

بعد تعديل مزودي النماذج في `~/.anda/config.yaml` ، قم بتحديث النماذج من المشغل أو اللوحة الجانبية للمتصفح، أو قم بتشغيل:

```bash
anda models reload
```

بعد تغيير إعدادات daemon التي لا تزال تتطلب إعادة تشغيل، اكتب:

```text
/reload
```

## أوامر الشرطة المائلة (Slash Commands)

| الأمر | متى يجب استخدامه |
| --- | --- |
| `/goal رسالة` | ضع هدفاً في حلقة المهام طويلة المدى حتى يقرر المدقق اكتماله أو حاجته إلى المزيد من العمل. |
| `/loop رسالة` | قم بتشغيل سؤال أو أمر شرطة مائلة بشكل متكرر. صِف الفاصل الزمني باللغة الطبيعية إذا كنت ترغب في ذلك، أو احذفه لترك النموذج يحدد سرعته بنفسه. |
| `/side رسالة` أو `/btw رسالة` | ابدأ مهمة جانبية للبحث أو الفحص أو العمل الذي لا ينبغي أن يقاطع المحادثة الرئيسية. |
| `/skill اسم-المهارة رسالة` | اطلب من مهارة (Skill) معينة لبيئة التشغيل التعامل مع الطلب. |
| `/steer رسالة` | قم بمقاطعة استدعاء الأداة التالي ووجه الاستجابة الحالية بتعليمات جديدة. |
| `/stop رسالة` | قم بإيقاف المهمة الحالية وترك المحادثة خاملة؛ ويمكن أن تتضمن الرسالة سبباً لذلك. |
| `/cancel رسالة` | إلغاء جلسة المحادثة النشطة؛ ويمكن أن تتضمن الرسالة سبباً لذلك. |

أمثلة:

```text
/goal إنهاء التحضير لهذا الإصدار: تشغيل الاختبارات، والتحقق من سجل التغييرات، والتحقق من وثائق سكريبت التثبيت، وتلخيص المخاطر.
/loop كل 5 دقائق /side تحقق من طابور النشر ولخص العقبات.
/side تحقق مما إذا كان ترقية هذه التبعية تشكل خطراً على التوافق.
/skill git-assistant قم بصياغة رسالة commit من التغييرات الحالية (diff).
/steer توقف عن إعادة الهيكلة وقم بإصلاح خطأ البناء أولاً فقط.
```

## أوامر CLI الشائعة

| الأمر | الغرض |
| --- | --- |
| `anda` | افتح واجهة مستخدم الطرفية التفاعلية. |
| `anda update` | قم بتحديث إصدار تم تثبيته بواسطة سكريبت التثبيت. |
| `anda status` | يوضح ما إذا كان daemon الخلفية قيد التشغيل. |
| `anda start` | ابدأ تشغيل daemon الخلفية. |
| `anda stop` | أوقف تشغيل daemon الخلفية. |
| `anda restart` | أعد تشغيل daemon الخلفية. |
| `anda models reload` | أعد تحميل مزودي النماذج من `config.yaml` دون إعادة تشغيل daemon. |
| `anda autostart status` | يوضح ما إذا كان بدء التشغيل التلقائي مسجلاً عند تسجيل الدخول. |
| `anda agent run --prompt "..."` | أرسل طلباً واحداً وانتظر النتيجة الكاملة دون فتح واجهة المستخدم. |
| `anda voice --record-secs 8` | ابدأ الإدخال الصوتي؛ يتطلب تكوين الترانسكريبت (transcription). |

يتطلب تشغيل الصوت أيضاً تعيين `tts.enabled: true`. إذا كنت تريد الإدخال الصوتي ومخرجات النص فقط، فأضف `--no-playback`.
