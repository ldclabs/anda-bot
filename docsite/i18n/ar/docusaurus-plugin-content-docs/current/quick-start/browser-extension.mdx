# امتداد المتصفح

يفتح امتداد المتصفح Anda Bot واجهة Anda في اللوحة الجانبية الأصلية للمتصفحات القائمة على Chromium — Google Chrome و Microsoft Edge. وهو يمثل جسراً إلى daemon Anda المحلي الذي يعمل على جهاز الكمبيوتر الخاص بك، بحيث يظل سياق المتصفح، وأدوات الطرفية، وملفات المشروع، والمهارات (Skills)، والوكلاء الفرعيون، والقنوات، وذاكرة Brain في بيئة تشغيل واحدة.

## المتطلبات

- متصفح Google Chrome أو Microsoft Edge 116 أو أحدث (أو أي متصفح قائم على Chromium).
- تثبيت Anda وإعداده مع مفتاح API واحد على الأقل لمزود النموذج.
- تشغيل daemon Anda المحلي على الكمبيوتر الذي تتحكم فيه.

قم بتثبيت الامتداد من متجر إضافات متصفحك:

- [Anda Bot على سوق Chrome الإلكتروني](https://chromewebstore.google.com/detail/anda-bot/injpfajmddchcphfkdkiflfddmajglfd) — لمتصفح Google Chrome ومتصفحات Chromium الأخرى.
- [Anda Bot على Microsoft Edge Add-ons](https://microsoftedge.microsoft.com/addons/detail/anda-bot/hljillhnmfbobihkehdlpmhbmdgophah) — لمتصفح Microsoft Edge.

يعمل الامتداد وخطوات الإعداد نفسها في كلا المتصفحين.

## توصيل اللوحة الجانبية

ابدأ تشغيل daemon المحلي:

```bash
anda start
```

قم بتوليد رمز متصفح:

```bash
anda browser token --days 365
```

انسخ كلاً من Gateway URL و Bearer token المطبوعين. افتح اللوحة الجانبية لـ Anda Bot في Chrome أو Edge، وافتح الإعدادات، ثم الصق القيمتين واحفظهما. بمجرد نجاح الاتصال، يمكنك التحدث مع الوكيل المحلي نفسه من أي صفحة ويب.

## العمل من صفحة ويب

عند إرسال رسالة من اللوحة الجانبية، يتلقى Anda عنوان الصفحة الحالية والرابط (URL) الخاص بها كسياق للطلب. إذا طلبت عملاً في المتصفح، يمكن للوكيل استخدام إجراءات المتصفح لفحص الصفحات وتشغيل التبويبات.

أمثلة مفيدة:

- تلخيص أو مقارنة الصفحة التي تقرأها حالياً.
- جمع المعلومات عبر علامات تبويب متعددة والاحتفاظ بروابط المصدر المرفقة.
- تعبئة النماذج، أو النقر على الأزرار، أو التمرير، أو التقاط لقطات الشاشة، أو تصدير الصفحة إلى ملف PDF عندما تتطلب المهمة ذلك.
- إعداد أوامر shell أو فحص ملفات المشروع المحلية مع الحفاظ على سياق صفحة المتصفح.
- وضع علامة مرجعية على رسائل المساعد المهمة، وتنظيمها في مجلدات، والعودة إلى المحادثة المصدر لاحقاً.

## قدرات المتصفح

يعرض الامتداد أدوات المتصفح لـ daemon المحلي. تتضمن الإجراءات الشائعة ما يلي:

- عرض التبويبات، والتبديل بينها، وفتحها، وإغلاقها، وإعادة تحميلها، والتنقل فيها.
- قراءة النصوص المحددة، واستخراج نص الصفحة، والتقاط لقطات الشاشة، وفحص بيانات إمكانية الوصول، وطباعة الصفحات إلى PDF.
- النقر، والكتابة، والضغط على المفاتيح، والتمرير، واختيار القوائم المنسدلة، ورفع الملفات، والتعامل مع مربعات الحوار.
- تنزيل الملفات وفحص التنزيلات التي بدأها المتصفح.

تستهدف إجراءات الصفحة والإدخال علامة التبويب النشطة. إذا كانت المهمة تحتاج إلى علامة تبويب أخرى، فاطلب من Anda التبديل بين علامات التبويب أولاً.

لفتح الملفات أو المجلدات المحلية باستخدام `open_file` ، قم بتمكين `السماح بالوصول إلى عناوين URL للملفات` (Allow access to file URLs) في صفحة تفاصيل ملحق المتصفح لـ Anda Bot (في متصفح Edge، افتح `edge://extensions` ← Anda Bot ← تفاصيل/Details).

## بيئة التشغيل المحلية والبيانات

يقوم الامتداد بتخزين Gateway URL و Bearer token ومعرف جلسة المتصفح والإعدادات في التخزين المحلي للمتصفح. ويرسل الأسئلة والملحقات المحددة وبيانات تعريف الجلسة ومعلومات علامة التبويب ذات الصلة إلى daemon Anda الذي تتصل به.

قد يقوم daemon ومزودو النماذج الذي تم تكوينهم في إعدادات Anda بمعالجة السؤال وبيانات المهمة. استخدم الامتداد فقط مع daemon وتكوين نموذج تثق به، خاصة على الصفحات الحساسة.

## القنوات الخارجية

اللوحة الجانبية للمتصفح هي واجهة واحدة فقط. إذا قمت بتكوين قنوات WeChat أو Telegram أو Discord أو Lark / Feishu، فيمكن لهذه الرسائل الوصول إلى daemon المحلي نفسه أثناء استمرار تشغيله على جهاز الكمبيوتر الخاص بك.

راجع [القنوات والصوت](../runtime/channels) لإعداد القناة.

## استكشاف الأخطاء وإصلاحها

- إذا لم تتمكن اللوحة الجانبية من الاتصال، فتأكد من أن `anda start` لا يزال يعمل وأن Gateway URL و Bearer token تم لصقهما تماماً كما هما.
- إذا فشلت إجراءات المتصفح على صفحات محمية مثل روابط `chrome://` أو `edge://` أو متجر إضافات المتصفح، فافتح صفحة ويب عادية. تمنع متصفحات Chromium تشغيل سكريبتات الامتداد على الصفحات المحمية.
- إذا ظهر خطأ في النموذج، فتحقق من مفتاح مزود النموذج في `~/.anda/config.yaml` أو متغير البيئة المستخدم لبدء تشغيل daemon.
